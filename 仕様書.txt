1. システム概要
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

飲食店向けのボトルキープ管理システム。
実際の顧客（飲食店オーナー・お客さん）に実際に使ってもらうことを
最終目標とする。ハッカソン出展はその第一歩。

【コア設計思想】
NFCタグのハードウェアUID（タグ固有のシリアル番号）を
動的にボトル情報へマッピングする。

  - タグへの個別書き込み作業ゼロ（箱から出してそのまま使う）
  - ボトルタグにはURLも何も書かない。UIDだけ使う。
  - ポータルNFCだけ店舗固有URLを書き込む（1店舗1枚）

【用語定義】
  NFC UID   : タグのハードウェア固有番号（例: 04:A3:2B:11:FF:C2）
              変更・上書き不可。スマホのNFC機能で読み取れる。
  ポータルNFC: お店の入口や卓上に置く「玄関タグ」。
              /my-bottles へのURLが書き込まれている。
  ボトルNFC  : 実際のボトルに取り付けるタグ。
              何も書かない。UID読み取りのみ利用。
  Web NFC API: Android版Chromeに搭載されているブラウザ機能。
              ブラウザからNFCタグのUIDを直接読み取れる。
  パスキー   : WebAuthn/FIDO2に基づく生体認証。
              顔認証・指紋でログイン。パスワード不要。

2. リリース戦略（2フェーズ）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Phase 1: ハッカソン出展版（Webアプリ）
    対象端末  : Android Chrome
    NFC方式   : Web NFC API（ブラウザ直接）
    フロント  : Leptos (WASM)
    認証      : パスキー（主）+ マジックリンク（フォールバック）
    優先実装  : コア機能のデモが動くこと

  Phase 2: 本番リリース版（ネイティブアプリ追加）
    対象端末  : iOS + Android
    NFC方式   : React Native（react-native-nfc-manager）
    バックエンド: Phase 1のAxum APIをそのまま流用 ← 変更不要
    追加実装  : AppStore/Play Store申請、Push通知
    ※ バックエンドを変えずにフロントだけ追加する設計

    - 操作端末: Android Chrome（Phase 1）/ iOS + Android（Phase 2）

  [システム管理者]
    - 新規店舗のセットアップ
    - ハッカソンスコープでは開発者が直接DBに投入で代替

4. NFCタグの種類と設定
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [ポータルNFC]  （各店1〜数枚）
    書き込み内容: https://yourapp.com/my-bottles
    設置場所    : 卓上、カウンター、入口の目立つ場所
    役割        : お客さんがアプリに入るための玄関口

  [ボトルNFC]    （店の在庫分、大量）
    書き込み内容: 何も書かない ← ここが最大の特徴
    取り付け方  : ボトルの首にかける or ラベル下に貼る
    役割        : UID = ボトルの"名札"
                  箱から出してそのまま使える。事前作業ゼロ。

5. ユーザーフロー
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

─────────────────────────────────
5-1. 店員: 新規ボトル登録フロー
─────────────────────────────────

  1. /admin を開く → PINコードを入力（店舗ごとに異なる）
  2. 「新規ボトル登録」ボタンをタップ → 登録モード開始
  3. 画面全体がパルスアニメーション「タグをかざして」
     → 内部でWeb NFC APIが scan() を開始
  4. ボトルのNFCタグを端末にかざす → UID取得
  5. 登録フォームが表示:
       - お客様名（例: 田中様）
       - お酒の銘柄（例: 山崎12年）
       - キープ期限（デフォルト: 3ヶ月後）
  6. 「登録」ボタン → DBに保存 → 「登録完了」

─────────────────────────────────
5-2. 店員: 日常運用（残量更新）
─────────────────────────────────

  1. 管理画面でボトル一覧から選ぶ か、
     ボトルのタグをかざす → ボトル情報が開く
  2. 残量を更新 → 保存

─────────────────────────────────
5-3. お客さん: 初回登録フロー
─────────────────────────────────

  1. 卓上のポータルNFCをタップ → /my-bottles が開く
  2. ログイン画面へ（未ログインの場合）
  3. パスキーで新規登録（メアド・ニックネームは任意）
     または メールリンクで登録（パスキー非対応端末）
  4. /my-bottles に戻る
  5. 「ボトルを追加」→ 自分のボトルNFCをかざす
  6. 自動でマイボトルにリンク → 完了

  ※ お客さんがやることはこれだけ。店員のメアド管理は不要。

─────────────────────────────────
5-4. お客さん: 2回目以降
─────────────────────────────────

  1. ホーム画面のアイコンをタップ（PWA）→ /my-bottles
  2. パスキー（顔認証・指紋）→ 即ログイン
  3. 全店舗のマイボトル一覧が表示

7. 認証設計
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [お客さんの認証]

  初回:
    A) パスキーで新規登録（推奨）
       → メアド・ニックネームは任意（通知が欲しい場合のみ）
       → 顔認証・指紋を登録 → セッション発行
    B) マジックリンクで登録（パスキー非対応端末）
       → メアドを入力 → リンクをタップ → セッション発行

  2回目以降:
    パスキー登録済み → 顔認証・指紋 → 即ログイン

  [スタッフの認証]

  店舗PINコード（4〜6桁）を入力 → staff_session Cookie発行（7日間）
  PINは shops テーブルで店舗ごとに管理。デフォルト: '1234'

  将来（Phase 2）:
    店舗オーナーがアカウント作成 → 店員をメール招待
    → 店員がパスキー設定 → ロール（staff）でアクセス制御

  [セキュリティ方針]
    - セッショントークン: httpOnly Cookie（XSS耐性）
    - HTTPS必須（本番）
    - パスキー: WebAuthn Level 2（FIDO2準拠）
    - RP_ID: localhost（開発）/ 本番ドメイン（本番）
    - LocalStorage: 認証情報は一切保存しない

10. UIデザイン原則（酔っ払いでも使えるUI）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [絶対に守るルール]
    ✓ ボタンの最小サイズ: 高さ64px以上（親指の幅）
    ✓ フォント: 最小18px、重要なものは24〜32px
    ✓ ダークモード標準（バーは暗い環境）
    ✓ 1画面1アクション
    ✓ タップエリアを広く（余白を削らない）
    ✓ 成功: 大きなアニメーション + 自動遷移
    ✓ エラー: 赤く・大きく・一言で（「もう一度かざして」）
    ✗ スワイプゲスチャーは使わない（酔うと誤操作）

  [カラースキーム]
    背景      : #0F0C16（深いダーク）
    アクセント : #F5A623（ゴールド）
    成功      : #4AC894（グリーン）
    エラー    : #FF9FAA（レッド）
    テキスト  : #F0EAE0（オフホワイト）

12. 制約事項・注意点
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ⚠️  Phase 1: iOS（iPhone/iPad）はNFC操作不可
      → Phase 2でReact Nativeアプリを作って対応

  ⚠️  Web NFC APIはHTTPS必須（+ localhost）
      本番デプロイ時はSSL証明書が必要（Let's Encrypt推奨）

  ⚠️  パスキーもHTTPS必須
      開発時は http://localhost:3000 でアクセスすること
      http://127.0.0.1:3000 ではパスキーが動作しない

  ⚠️  メール送信サーバー
      マジックリンク認証にSMTP設定が必要
      環境変数: SMTP_HOST, SMTP_USER, SMTP_PASS, SMTP_FROM
